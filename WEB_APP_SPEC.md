## 1. Web アプリのコンセプト・スコープ

### 1-1. ポジショニング
iOSアプリが「**記録する／投稿する**」現場ツールであるのに対し、
Web は「**じっくり見る／計画する**」ための大画面・PC/タブレット向けコンパニオン。

| | iOSアプリ | Web アプリ（本仕様） |
|---|-----------|----------------------|
| スポット投稿 | ✅ | ❌（閲覧のみ） |
| ルート記録（GPS） | ✅ | ❌（閲覧のみ） |
| スポット閲覧・検索 | ✅ | ✅ |
| ルート閲覧 | ✅ | ✅ |
| ツーリングプラン作成・編集 | ✅ | ✅（中核機能） |
| 実績・統計の閲覧 | ✅ | ✅ |

### 1-2. スコープ定義
- **In Scope（やる）**: ログイン認証、スポット閲覧、ルート閲覧、ツーリングプラン管理、実績/統計閲覧、ユーザープロフィール閲覧、ブログ
- **Out of Scope（やらない）**: **新規会員登録（アプリのみで実施）**、スポット新規投稿、写真アップロード投稿、GPS ルート記録、課金（StoreKit / アプリ内課金）

## 3. 不足機能の洗い出し（Gap Analysis）

iOS（`APP_FEATURES.md`）に対し、**Web のスコープ（閲覧＋プラン）内で不足している**機能。
スコープ外（投稿・記録・課金）は対象外。優先度：**高=体験の根幹 / 中=価値向上 / 低=あると良い**。

### 4-2. ページ別 目標仕様（主要差分のみ）

#### 発見 `/index.html`（強化）
- 既存：地図/リスト、検索、地域フィルタ、クラスタ、プラン追加

- 既存：写真・地図・投稿者・住所・プラン追加

#### ルート探索 `/routes.html`（新規）
- `shared_routes` を一覧＋地図で表示、新着/人気/距離/エリアで絞り込み（R1）

#### ルート詳細 `/route-detail.html`（強化）

### 4-3. 認証・権限モデル
- Firebase Auth（メール/パスワード）。iOS とアカウント共通。**新規登録はアプリのみ**で行い、Web は既存アカウントのログイン専用。
- 公開ページ（発見・詳細・ルート・ユーザー・ブログ）は**未ログインでも閲覧可**。
- 個人データ（プラン・実績・統計・アカウント）は**要ログイン**＋ Firestore セキュリティルールで `uid` 一致を担保。
- サブスク状態は Firestore（iOS が StoreKit 課金結果を保存している前提のフィールド）を**読み取り専用**で参照（X2/P1）。Web では課金処理を行わない。

| 項目 | 方針 |

| 多言語 | JA/EN（X1）。`lang` 切替 + 文言辞書 |
| 帰属表示 | OpenStreetMap/Google の Attribution をフッター等に明記（X5） |
| アーキ | 当面は現行踏襲（Vue 3 CDN + 1ファイル完結）。ページ増に伴い共通処理（Firebase 初期化・地図ユーティリティ）の切り出しを検討 |

## 6. 明確な対象外（Non-Goals）

以下は **Web では実装しない**（iOSアプリの役割）:
- **新規会員登録**（アプリのみで実施。Web は既存アカウントのログインのみ）
- スポットの新規投稿・写真アップロード投稿
- GPS によるルート記録・走行中のトラッキング
- アプリ内課金（StoreKit / サブスク購入）— Web は**状態の参照のみ**
- バックグラウンド GPS、オフライン地図キャッシュ等の端末ネイティブ機能

---

### 付録：本仕様の根拠（調査メモ）
- `login.html` は `signInWithEmailAndPassword` と `sendPasswordResetEmail` のみ実装（新規登録・退会・PW変更なし）。
- `detail.html` の「口コミ」は meta description の文言のみで、評価表示の実装はなし。
- 「道の駅」「michi」「subscription/isPro/isPlus」は全 HTML に出現せず（未対応）。
- 共有ルートは `user.html`（ユーザー別一覧）と `route-detail.html`（単体）からのみ到達可能で、横断的な探索ページは未実装。
- `touring-plan.html` は上限を `PLAN_LIMIT_FREE = 3` / `SPOT_LIMIT_FREE = 3` でハードコード。

### 付録：`shared_routes` 実データの構造（R1 実装時に本番Firestoreで確認）
- フィールド: `distance`（**メートル**）, `title`, `coverImageUrl`, `routeID`, `createdAt`（Timestamp）, `jsonUrl`, `jsonFileName`, `userID`。
- **`isPublic` は多くのドキュメントに存在しない**（=共有された時点で公開扱い）。`user.html` も `isPublic === false` のときだけ「非公開」と表示する実装。→ ルート探索は「全件取得し `isPublic===false` のみ除外」が正解（`where('isPublic','==',true)` だと0件になる）。
- **`title` は UUID やファイル名（`route_*.json`）が入っており人間可読でない**ケースが大半。→ `routes.html` では UUID/ファイル名を検知して「ツーリングルート」にフォールバック表示。
- ルート座標は `jsonUrl`（Storage 上の JSON）。標高・セグメント・マーカー写真（R2）を出すにはこの JSON 構造の確認が必要。
- 投稿者名・アイコンは `userInfo/{userID}` の `userName` / `userIcon`。
- 本番のルート総数は現状ごく少数（確認時点で9件）。ページネーションは実装済みだが当面はほぼ1ページ。
